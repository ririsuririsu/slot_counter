# Feature Specification: Slot Counter - Monkey Turn V Edition

**Feature Branch**: `main`
**Created**: 2025-12-06
**Status**: Draft
**Input**: スロット「モンキーターンV」の小役カウンターWebアプリ（PWA対応）

## Overview

パチスロ「モンキーターンV」専用の小役カウンターWebアプリケーション。既存のGoogle Apps Script版をReact + PWAで再構築し、オフライン対応・複数台管理・統計グラフ表示・データエクスポート機能を追加する。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 小役カウント (Priority: P1)

ユーザーは実機でプレイしながら、成立した小役をタップしてカウントできる。大きなボタンで片手操作でも使いやすく、即座にカウントが反映される。

**Why this priority**: アプリの核となる機能。これがなければ何も始まらない。

**Independent Test**: アプリを開き、任意の小役の+1ボタンをタップして数値が増加することを確認。-1ボタンで減少も確認。

**Acceptance Scenarios**:

1. **Given** アプリを起動した状態, **When** 「5枚役リリベ」の+1ボタンをタップ, **Then** カウントが1増加して表示される
2. **Given** カウントが5の状態, **When** -1ボタンをタップ, **Then** カウントが4に減少する
3. **Given** カウントが0の状態, **When** -1ボタンをタップ, **Then** カウントは0のまま（負の値にならない）

---

### User Story 2 - 総ゲーム数入力と確率計算 (Priority: P1)

ユーザーは総ゲーム数を入力し、5枚役の合算確率を確認できる。設定推測の参考にする。

**Why this priority**: 設定推測に必要な情報。カウントだけでは意味がない。

**Independent Test**: 総ゲーム数を入力し、5枚役確率が「1/XX.X」形式で表示されることを確認。

**Acceptance Scenarios**:

1. **Given** 5枚役合計が50回、**When** 総ゲーム数1000を入力, **Then** 「5枚役確率: 1/20.0」と表示される
2. **Given** 総ゲーム数入力画面, **When** ドラム式UIで数値を選択して更新, **Then** 確率が再計算される

---

### User Story 2.5 - 設定判別（二項分布） (Priority: P1)

ユーザーは5枚役確率から各設定の可能性を二項分布で計算し、設定推測の参考にできる。

**Why this priority**: 単なる確率表示だけでなく、各設定への確率を算出することで設定推測の精度が上がる。

**Independent Test**: 総ゲーム数と5枚役合計を入力し、各設定（1,2,4,5,6）の確率が%表示されることを確認。

**Acceptance Scenarios**:

1. **Given** 5枚役合計50回・総ゲーム数1305, **When** 設定判別を表示, **Then** 各設定の二項分布確率が表示される（例：設定1: 0.0%, 設定6: 50.0%）
2. **Given** 設定判別表示中, **When** 5枚役カウントが増加, **Then** 自動的に確率が再計算される
3. **Given** 5枚役が0回, **When** 設定判別を表示, **Then** 設定1が100%と表示される（低設定ほど5枚役が出にくい）

---

### User Story 3 - データ永続化とオフライン対応 (Priority: P1)

ユーザーはオフライン環境でもアプリを使用でき、データはローカルに保存される。電波の悪いパチンコ店内でも問題なく動作する。

**Why this priority**: パチンコ店内は電波が悪いことが多く、オフライン対応は必須。

**Independent Test**: 機内モードでアプリを起動し、カウント操作ができることを確認。

**Acceptance Scenarios**:

1. **Given** オンラインでアプリを一度開いた後, **When** オフラインで再度アクセス, **Then** アプリが正常に動作する
2. **Given** カウントを更新した状態, **When** ブラウザを閉じて再度開く, **Then** データが保持されている

---

### User Story 4 - 複数台管理 (Priority: P2)

ユーザーは同じ日に複数台を打った場合、台ごとにデータを管理できる。台番号やメモを付けて区別する。

**Why this priority**: 1日に複数台打つことがあり、台ごとの比較が必要。

**Independent Test**: 新規台を追加し、それぞれ独立してカウントできることを確認。

**Acceptance Scenarios**:

1. **Given** 台1でカウント中, **When** 「新規台追加」をタップ, **Then** 台2が作成され、カウントが0からスタート
2. **Given** 複数台が存在, **When** 台リストから台1を選択, **Then** 台1のデータが表示される
3. **Given** 台を選択中, **When** 台名や台番号を編集, **Then** 変更が保存される

---

### User Story 5 - 履歴記録 (Priority: P2)

ユーザーは総ゲーム数入力時に履歴が記録され、時系列で確率の推移を確認できる。

**Why this priority**: 打ち続けるかどうかの判断材料になる。

**Independent Test**: 複数回ゲーム数を入力し、履歴一覧に記録が増えることを確認。

**Acceptance Scenarios**:

1. **Given** ゲーム数1000で更新, **When** 履歴を確認, **Then** タイムスタンプ・ゲーム数・5枚役確率が記録されている
2. **Given** 複数の履歴がある, **When** 履歴一覧を表示, **Then** 時系列順に表示される

---

### User Story 6 - 統計グラフ表示 (Priority: P2)

ユーザーは5枚役確率の推移をグラフで視覚的に確認できる。

**Why this priority**: 数値だけでなくトレンドを把握したい。

**Independent Test**: 履歴データがある状態でグラフ画面を開き、折れ線グラフが表示されることを確認。

**Acceptance Scenarios**:

1. **Given** 履歴データが5件以上, **When** グラフ画面を開く, **Then** 5枚役確率の推移が折れ線グラフで表示される
2. **Given** グラフ表示中, **When** 特定のポイントをタップ, **Then** その時点の詳細情報が表示される

---

### User Story 7 - データエクスポート (Priority: P3)

ユーザーは収集したデータをCSVまたはJSON形式でエクスポートし、外部で分析できる。

**Why this priority**: 長期的な分析や他ツールとの連携に必要。

**Independent Test**: エクスポートボタンをタップし、ファイルがダウンロードされることを確認。

**Acceptance Scenarios**:

1. **Given** データがある状態, **When** CSVエクスポートをタップ, **Then** CSVファイルがダウンロードされる
2. **Given** データがある状態, **When** JSONエクスポートをタップ, **Then** JSONファイルがダウンロードされる

---

### User Story 8 - リセット機能 (Priority: P1)

ユーザーは台を離れる時やデータをクリアしたい時に、全カウンターと履歴をリセットできる。

**Why this priority**: 新しい台を始める時に必須。

**Independent Test**: リセットボタンをタップし、確認後に全データがクリアされることを確認。

**Acceptance Scenarios**:

1. **Given** カウントや履歴がある状態, **When** リセットボタンをタップ, **Then** 確認ダイアログが表示される
2. **Given** 確認ダイアログ表示中, **When** 「はい」をタップ, **Then** 全カウンターが0に、履歴がクリアされる
3. **Given** 確認ダイアログ表示中, **When** 「キャンセル」をタップ, **Then** データは保持される

---

### Edge Cases

- 総ゲーム数が0の時に確率計算しようとした場合 → エラー表示またはN/A表示
- 5枚役合計が0の時の確率計算 → 「1/∞」または「-」表示
- ストレージ容量が満杯の時 → 警告を表示し、古いデータの削除を促す
- 非常に長いセッション（10000ゲーム以上）でのパフォーマンス
- 複数タブで同時に開いた場合のデータ競合

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST 小役ごとにカウント（+1/-1）ができる
- **FR-002**: System MUST カウントが0未満にならないよう制御する
- **FR-003**: System MUST 総ゲーム数を入力できる（4桁のドラム式UI）
- **FR-004**: System MUST 5枚役（リリベ・リベべ・リリボ）の合算確率を計算・表示する
- **FR-004.5**: System MUST 二項分布を用いて各設定（1,2,4,5,6）の確率を計算・表示する
- **FR-005**: System MUST データをlocalStorageに永続化する
- **FR-006**: System MUST PWAとしてオフラインで動作する
- **FR-007**: System MUST 複数台のデータを管理できる
- **FR-008**: System MUST 履歴を記録する（タイムスタンプ、ゲーム数、5枚役合計、確率）
- **FR-009**: System MUST 全データをリセットできる（確認付き）
- **FR-010**: System SHOULD 5枚役確率の推移をグラフ表示する
- **FR-011**: System SHOULD データをCSV/JSON形式でエクスポートできる
- **FR-012**: System MUST モバイルファーストのレスポンシブデザインである
- **FR-013**: System MUST 大きなボタンで片手操作しやすいUIを提供する

### Key Entities

- **Machine（台）**: 台番号、台名、作成日時、最終更新日時
- **Counter（カウンター）**: 小役名、カウント値、背景色
- **History（履歴）**: タイムスタンプ、総ゲーム数、5枚役合計、確率
- **KoyakuDefinition（小役定義）**: 小役名、表示順、背景色、カテゴリ（設定差あり/なし）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがカウント操作を1秒以内に完了できる
- **SC-002**: オフライン状態でも全機能が動作する
- **SC-003**: ページ読み込みが3秒以内（初回）、1秒以内（キャッシュ後）
- **SC-004**: モバイル端末（iPhone SE以上）で快適に操作できる
- **SC-005**: 1000件以上の履歴データでもパフォーマンスが低下しない

## Non-Functional Requirements

- **NFR-001**: React + TypeScript で構築
- **NFR-002**: PWA対応（Service Worker、Web App Manifest）
- **NFR-003**: localStorage でデータ永続化
- **NFR-004**: Lighthouse Performance Score 90以上
- **NFR-005**: 将来的に他機種（激走チェイサー等）を追加できる拡張性

## Out of Scope (v1.0)

- ユーザー認証・クラウド同期
- 機種間のデータ比較
- 他ユーザーとのデータ共有
- 複数機種の同時管理（v1.0はモンキーターンV専用）
