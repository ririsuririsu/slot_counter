# 北斗の拳 転生の章2 — 伝承モード推測補助ツール 調査資料

> 既存の `hokuto-tensei2-spec.md`（網羅資料）から **伝承モード周辺だけを切り出し、推測ロジック設計に必要な数値と観測モデル**に絞った資料。

## 1. 立ち位置

- 既存実装（詳細ログ方式）と**並列に存在するサブツール**として実装する。
- データモデル方針: **同一機種(`HokutoMachine`)内でタブ分離**。`hokuto`データは無関係に保ち、`denshoHelper`データブロックを別領域に持つ。
- 操作タブを切り替えながら実戦で使うことを想定。両者のログ・推定は完全独立。

## 2. 伝承モード基礎仕様（確定値）

### 2.1 3レベルのループ仕様（全設定共通）

| レベル | 継続率(per ハズレ・リプレイ) | 転落率 | 平均ループ回数 |
|---|---|---|---|
| ショート | 75.5% | **32.8%** | 約2〜3回 |
| ミドル | 95.3% | **4.7%** | 約5回 |
| ロング | 98.8% | **1.2%** | 約10回 |

> ※「継続率」と書かれている数値は元解析記事では「天破ループ率」(24.5%/69.4%/90.1%)として表記されているが、これは**1サイクル(=次の天破当選)までの継続率**。本ツールが扱う「ハズレ/リプレイ1回あたりの非転落率」は `1 - 転落率` = 0.672 / 0.953 / 0.988 になる（元設計書の値と一致）。

### 2.2 転落抽選の対象役

- **ハズレ・リプレイ成立時のみ**転落抽選
- 右下がりベル・レア役での転落は **発生しない**
- ※ 通常時の状態転落抽選とは別の仕組み（混同注意）

### 2.3 伝承突入条件

| 契機 | ショート | ミドル | ロング |
|---|---|---|---|
| 天破の刻終了後（必ず突入） | 98.4% | 1.2% | 0.4% |
| 朝一リセット時 | 6.3% | — | — |
| AT終了時（非伝承中） | 1.6% | — | — |
| 設定変更時 | 同上の朝一リセット扱い | — | — |

> ※ 伝承中レア役では昇格抽選あり（具体的な数値は非公開、観測で補正）。
> ※ 伝承中AT当選後は **同一伝承モードから再スタート**（天撃獲得時除く）。

### 2.4 伝承中の天破当選率（全設定共通）

| 役 | 当選率 |
|---|---|
| ハズレ | 5.5% |
| リプレイ | 5.5% |
| 右下がりベル | 15.6% |
| 弱チェリー | 25.0% |
| スイカ | 33.6% |
| チャンス目 | 75.0% |
| 勝舞揃い | 75.0% |
| 強チェリー | 100% |
| 確定チェリー | 100% |

### 2.5 滞在示唆

- 宝玉ランプ（あべしカウンタ左下）
  - **白点滅** = 伝承ミドル以上 or AT本前兆
  - **紫点滅** = 伝承ロング濃厚 or AT本前兆
- カサンドラステージ = 高確 or 伝承モード滞在示唆
- AT本前兆と区別がつかない点に注意（推定の誤り原因になる）

## 4. 推測ロジック設計 — 物理量ベースのベイズ更新

> 元設計書のヒューリスティック係数（×0.80, ×0.70 等）は、**物理量から導出可能**。経験則固定ではなく確率モデルとして設計する。

### 4.1 伝承滞在確率 `P(densho)` のベイズ更新

各観測 E（役の成立とその結果）について:

```
P(densho | E) = P(E | densho) * P(densho) / P(E)
```

#### 役×結果ごとの尤度比 `LR = P(E|densho) / P(E|¬densho)`

通常状態(low/normal/high の混合)での天破当選率 `r_normal` は概ね 1〜10% 程度。簡略化のため低確基準で計算した参考LR:

| 観測E | P(E\|densho) | P(E\|¬densho)≈低確 | LR |
|---|---|---|---|
| 弱チェ成立&天破当選 | 0.250 | 0.008 | 31.3 |
| 弱チェ成立&非当選 | 0.750 | 0.992 | 0.756 |
| スイカ成立&非当選 | 0.664 | 0.992 | 0.669 |
| チャンス目成立&非当選 | 0.250 | 0.980 | 0.255 |
| 強チェ成立&非当選 | 0.000 | 0.750 | 0 |
| ハズレ/リプレイ後 (※転落) | 後述 | — | — |

→ 元設計書の `×0.80, ×0.70, ×0.25, =0` 係数はおおむね物理値と一致しており、**理論的に正当化できる**ことが判明。

#### ハズレ・リプレイの扱い（転落モデル）

レベル不明時の伝承継続確率は3レベルの加重和:

```
denProb_next = P(short)*0.672 + P(middle)*0.953 + P(long)*0.988
```

レベル事後確率も同時に更新する（後述）。

### 4.2 伝承レベル `P(short|middle|long)` の推定

突入直後の事前分布: `{short: 0.984, middle: 0.012, long: 0.004}`（天破経由時）

#### 観測ごとの更新

- **天破当選回数(=ループ回数) `n`**: 各レベルが現在も滞在している尤度は `継続率^n`（厳密にはハズレ/リプレイ回数 `k` で `(1-転落率)^k`）
- **非天破イベント数 `k`(ハズレ・リプレイのみカウント)**: 同上
- **早い再天破 (10G以内)**: long の事後確率上昇（既存 ratio で補正）
- **長期ハマり (30G以上ハズレ/リプレイ連続)**: short の事後低下（生き残ってる確率が低いため）

> 元設計書の "long +3" 等の加点は、対数尤度比の近似と解釈可能。そのまま使うか、log-likelihood ベースで明示的に書き直すか選択。

### 4.3 観測ウィンドウ → 状態スコア

伝承終了後（denProb < 閾値 or 強チェ非当選 等で確定）から最大30Gの観測ウィンドウを開く。

このウィンドウ中の観測:
- ステージ（カサンドラ→高確示唆、荒野→通常示唆、それ以外→低確示唆）
- レア役での反応（昇格演出 or 天破当選）
- 早期天破当選（=高確濃厚）
- 高確確定演出（LED白点滅）

これらから state ∈ {low, normal, high} の事後分布を更新する。

### 4.4 設定推定 — 既存テーブル直結

観測ウィンドウの出力 `state ∈ {low, normal, high}` をサンプルとして蓄積。

設定別の事後確率:

```
P(setting=s | samples) ∝ ∏_i P(state_i | setting=s)
                     = ∏_i DENSHO_END_TRANSITION[s][state_i]
```

→ 元設計書の「score = high*2 + normal*1 - low*2」は表示用ヘルパーとして残し、**内部推定はベイズで実装**。

### 4.5 サンプル数目安

| サンプル数 | 信頼度 |
|---|---|
| 5回 | 参考程度 |
| 10回 | 判断可能（設定1↔6の判別が概ね可能） |
| 15回以上 | 高信頼 |

## 5. UIに必要な入力イベント（ドラフト）

### 必須入力（手動）

- **伝承モード開始**（天破終了の認識） — 自動or手動
- **天破当選**（伝承中の継続観測）
- **レア役成立**（弱チェ/強チェ/スイカ/チャンス目/勝舞）+ 天破当否
- **ハズレ/リプレイの累計回数** — ゲーム数差分から自動推定可
- **観測ウィンドウ中のステージ変化**
- **高確確定演出**

### 推奨入力（任意）

- 宝玉ランプ点滅（白/紫）
- カサンドラ移行タイミング
- 観測ウィンドウ終了の手動確定

### 自動補完

- ハズレ・リプレイ回数: 「総ゲーム差 - レア役入力数 - 天破中ゲーム数 - リプレイ入力済」で算出（厳密でなくとも転落モデルへの寄与は近似可）
- 通常役での非当選イベント: 暗黙

## 6. 設計書からの修正点まとめ

| 元設計書 | 修正案 | 理由 |
|---|---|---|
| 時間減衰 ×0.90/G 等 | **削除** | 実機は per-game ではなく per-役 のマルコフ抽選 |
| 転落係数 0.672/0.953/0.988 | **保持** | 物理値と一致 |
| 役別係数 0.80/0.70/0.25 | **物理量から再導出** | 解析値ベースで妥当だが、テーブル化して保守可能に |
| score = high×2 + normal - low×2 | **表示用ヘルパー** | 内部推定はベイズ |
| score閾値での上下判別 | **設定別事後確率** | DENSHO_END_TRANSITION直結で精度向上 |
| レベル "long +3" 等の加点 | **対数尤度ベース** | または明示的なベイズ更新 |
| 高確確定演出 `low -= 2` | **min/maxクリップ or 確定証拠化** | スコア負値の扱い未定義回避 |
